# AGENTS

## Project overview
This repository contains Teal source code and tests for inmation-related types and utilities. The workflow relies on Teal type-checking and build outputs generated by Cyan/Froge.

## Build and run
- Build the project: cyan build
- Build and run a script: froge build run
- Run a script (auto-bundles): froge run test/run.lua
- Bundle a script: froge bundle test/run.lua

## Tests
- Run all tests: froge test
- Run a specific test file: froge test test/name.lua
- Run a specific test by name: froge test test_name
- Build + run a test file: cyan build && froge test test/name.lua

## Teal language rules and conventions
### Types vs values
- Types are declared with local type, local interface, or local record.
- Values are declared with local name or local function.
- Types are compile-time only and cannot be used as runtime values.

### Importing types vs values
- Import types via a module and local type alias:
  - local Module = require("Module")
  - local type MyType = Module.MyType
- Import values via a module value:
  - local f = Module.functionName

### Records vs instances
- record defines a type, not a constructor.
- Use a constructor function to create instances.
- Do not call methods on record types directly.

### Type checking
- Use value is Type to narrow types at runtime.

### Union types
- Use unions like string | number | boolean and narrow with is checks.

### Function return types
- Functions that return nil should declare : nil in the signature.
- If a function has a : nil return type, ensure it explicitly returns nil at the end and on early exits.
- If no return type is specified in the interface, avoid adding : nil in the implementation.

### Type casting
- Use value as Type for casting.
- Use value as (Type1, Type2) for multi-return casts.
- Use value as any only when necessary for untyped external APIs.

### Path operations
- Use path:absolutePath() to get a string.
- Concatenating paths requires converting the IPath to string first.

### Common fixes applied
1. Type imports: use local type for imported types.
2. Constructor calls: use Module.constructor() not Type.constructor().
3. Function return types: enforce explicit return nil for : nil functions.
4. Type casting: use as any for syslib properties that lack types.
5. Path operations: use path:absolutePath() for concatenation.
6. Variable declarations: declare typed locals before assignment.
7. Arrays: cast any results to typed arrays as needed.

## Session-specific notes
- When adding new interfaces, register them in dev-teal/src/inmation/interfaces/inmation-interfaces.d.tl.
- Use Appendable as a shared helper for add/appendable patterns where appropriate.
- Namespace is a conceptual container (not a real inmation object) for creating children under a path.
- Mass operations require syslib.model.codes.MassOp.* values; ensure syslib.d.tl is complete (e.g., UPDATE).
